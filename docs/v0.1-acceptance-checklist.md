# Icoo v0.1 验收清单

本文档把 `v0.1` 的最终验收拆成两部分：

1. `icoo` 本体交付验收
2. `proxy` 可用交付验收

目标是把发版前需要执行的关键命令固定下来，避免验收再次依赖口头记忆。

相关文档：

- 总交付规划：`docs/v0.1-delivery-plan.md`
- v0.1 发布说明：`docs/v0.1-release-notes.md`
- proxy 可用交付说明：`examples/proxy/v0.1-delivery.md`

## 一、`icoo` 本体交付验收

### 1. 全量测试

```powershell
$env:GOFLAGS='-p=1'
go test ./...
```

通过标准：

- 全部测试通过
- 不因默认并发导致资源/内存放大问题

### 2. 项目检查

```powershell
go run ./cmd/icoo check examples/proxy
```

通过标准：

- 返回 `ok: ...examples/proxy/src/main.ic`
- project 模式可正确识别入口

### 3. bundle 验收

```powershell
go run ./cmd/icoo bundle examples/proxy <temp>/proxy.icb
go run ./cmd/icoo inspect <temp>/proxy.icb
go run ./cmd/icoo extract <temp>/proxy.icb <temp>/extract
```

通过标准：

- bundle 成功生成
- inspect 能正确输出 `entry`、`entry_function`、`root_alias`
- proxy bundle 当前应只包含 8 个模块：
  - `lib/admin.ic`
  - `lib/auth.ic`
  - `lib/catalog.ic`
  - `lib/config.ic`
  - `lib/json_helpers.ic`
  - `lib/proxy_handler.ic`
  - `lib/state.ic`
  - `src/main.ic`
- 不再包含旧的 `lib/store.ic`

### 4. build 验收

```powershell
go run ./cmd/icoo build examples/proxy <temp>/proxy.exe
go run ./cmd/icoo inspect <temp>/proxy.exe
```

通过标准：

- 可执行文件生成成功
- inspect 能读取嵌入 bundle 元数据
- 模块列表与 bundle 验收一致

## 二、`proxy` 可用交付验收

### 1. 冒烟脚本

```powershell
go run ./cmd/icoo run examples/proxy/smoke.ic
```

通过标准：

- 脚本正常退出
- 输出包含：
  - `== proxy smoke ==`
  - `proxy status: 200`
  - `admin request total: 3`

### 2. 目标测试

```powershell
$env:GOFLAGS='-p=1'
go test ./pkg/api -run 'StdService|ExamplesRun/smoke' -v
```

通过标准：

- `TestExamplesRun/smoke` 通过
- `TestRuntimeRunSource_IteratesStdServiceModule` 通过
- `TestRuntimeRunSource_StdServiceMonitoring` 通过

### 3. 行为验收

proxy 冒烟和目标测试应共同覆盖以下行为：

- Bearer 鉴权
- 路由改写
- 模型别名改写
- 请求 ID 透传
- 上游转发
- `/healthz`
- `/readyz`
- `/admin/routes`
- `/admin/requests`
- `/admin/monitor`
- recent requests
- counters
- latency summary

### 4. 结构验收

当前 proxy 项目结构应满足：

- `app.ic` 为项目入口
- `src/main.ic` 暴露 `start` / `stop` / `main`
- 旧 `lib/store.ic` 不再作为主路径依赖

## 三、发版前最终复核

发版前建议按下面顺序再跑一遍：

1. `go run ./cmd/icoo check examples/proxy`
2. `go run ./cmd/icoo run examples/proxy/smoke.ic`
3. `$env:GOFLAGS='-p=1'; go test ./pkg/api -run 'StdService|ExamplesRun/smoke' -v`
4. `$env:GOFLAGS='-p=1'; go test ./...`
5. `bundle` / `inspect` / `extract`
6. `build` / `inspect`

当这些步骤全部通过时，可以认为：

- `icoo` 本体交付具备 `v0.1` 条件
- `proxy` 可用交付具备 `v0.1` 条件
